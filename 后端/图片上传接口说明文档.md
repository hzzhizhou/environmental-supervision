# 图片上传接口说明文档

***

## 📖 文档概述

本文档详细说明系统中的图片上传功能，包括头像上传和现场图片上传两种场景。

***

## 🌐 服务基础信息

| 属性               | 值                       |
| ---------------- | ----------------------- |
| **服务地址**         | `http://localhost:8080` |
| **认证方式**         | JWT Token（上传时需要）        |
| **Content-Type** | `multipart/form-data`   |

***

## 📋 图片上传接口列表

| 功能       | 接口                                   | 方法 | 是否需要认证 |
| -------- | ------------------------------------ | ---- | -------- |
| 上传头像    | `/api/profile/upload-avatar`         | POST | ✅ 是      |
| 上传现场图片  | `/api/supervisor/reports/upload-photo` | POST | ✅ 是      |
| 访问头像    | `/api/profile/avatar/{filename}`     | GET  | ❌ 否      |
| 访问现场图片  | `/api/file/report/{filename}`        | GET  | ❌ 否      |

***

## 1. 上传头像

### 请求信息

| 属性               | 值                            |
| ---------------- | ---------------------------- |
| **URL**          | `/api/profile/upload-avatar` |
| **Method**       | POST                         |
| **认证**           | 需要（携带 JWT Token）              |
| **Content-Type** | `multipart/form-data`        |

### 请求参数

| 字段     | 类型            | 必填 | 说明                       |
| ------ | ------------- | -- | ------------------------ |
| userId | Integer       | ✅  | 用户ID（表单字段）               |
| file   | MultipartFile | ✅  | 头像文件（支持 jpg/png/jpeg 格式） |

### 请求示例

```bash
POST /api/profile/upload-avatar
Content-Type: multipart/form-data
Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...

# 表单字段：
# userId: 1
# file: (选择本地图片文件)
```

### 响应示例

```json
{
    "code": 200,
    "message": "头像上传成功",
    "data": "/api/profile/avatar/1704758400000.jpg"
}
```

### 访问头像

上传成功后，可通过以下 URL 访问头像（无需认证）：

```
GET /api/profile/avatar/1704758400000.jpg
```

### 存储位置

```
项目根目录/uploads/avatar/1704758400000.jpg
```

| 字段   | 类型            | 必填 | 说明 |

POST /api/supervisor/reports/upload-photo

## 🔄 图片上传使用流程

### 场景一：上传头像

```
1. 用户登录 → 获取 JWT Token
           ↓
2. 调用 POST /api/profile/upload-avatar 上传头像
           ↓
3. 后端返回头像 URL: /api/profile/avatar/xxx.jpg
           ↓
4. URL 自动保存到用户表的 avatar_url 字段
           ↓
5. 前端通过 URL 显示头像
```

### 场景二：上报污染时上传现场图片

```
1. 用户登录 → 获取 JWT Token
           ↓
2. 调用 POST /api/supervisor/reports/upload-photo 上传现场图片（可多次调用）
           ↓
3. 后端返回图片 URL: /api/file/report/xxx.jpg
           ↓
4. 前端将所有图片 URL 用逗号分隔拼接
           ↓
5. 调用 POST /api/supervisor/reports 上报污染，将图片 URL 传入 photos 字段
           ↓
6. 图片 URL 保存到上报记录表的 photos 字段
```

## 📝 前端集成示例

### JavaScript/TypeScript 示例

```javascript
// 上传头像
async function uploadAvatar(userId, file) {
    const formData = new FormData();
    formData.append('userId', userId);
    formData.append('file', file);
    
    const response = await fetch('/api/profile/upload-avatar', {
        method: 'POST',
        headers: {
            'Authorization': 'Bearer ' + localStorage.getItem('token')
        },
        body: formData
    });
    
    const result = await response.json();
    return result.data; // 返回头像URL
}

// 上传现场图片
async function uploadPhoto(file) {
    const formData = new FormData();
    formData.append('file', file);
    
    const response = await fetch('/api/supervisor/reports/upload-photo', {
        method: 'POST',
        headers: {
            'Authorization': 'Bearer ' + localStorage.getItem('token')
        },
        body: formData
    });
    
    const result = await response.json();
    return result.data.url; // 返回图片URL
}

// 显示图片
function displayImage(url) {
    const img = document.createElement('img');
    img.src = url;
    document.body.appendChild(img);
}
```

### Vue.js 示例

```vue
<template>
    <!-- 上传头像 -->
    <input 
        type="file" 
        accept="image/jpeg,image/png"
        @change="handleAvatarUpload"
    />
    
    <!-- 上传现场图片 -->
    <input 
        type="file" 
        accept="image/*"
        multiple
        @change="handlePhotoUpload"
    />
    
    <!-- 显示图片 -->
    <img :src="avatarUrl" v-if="avatarUrl" />
    <img v-for="photo in photos" :key="photo" :src="photo" />
</template>

<script setup>
import { ref } from 'vue';

const avatarUrl = ref('');
const photos = ref([]);

const handleAvatarUpload = async (event) => {
    const file = event.target.files[0];
    const formData = new FormData();
    formData.append('userId', 1);
    formData.append('file', file);
    
    const response = await fetch('/api/profile/upload-avatar', {
        method: 'POST',
        headers: {
            'Authorization': 'Bearer ' + localStorage.getItem('token')
        },
        body: formData
    });
    
    const result = await response.json();
    avatarUrl.value = result.data;
};

const handlePhotoUpload = async (event) => {
    const files = Array.from(event.target.files);
    
    for (const file of files) {
        const formData = new FormData();
        formData.append('file', file);
        
        const response = await fetch('/api/supervisor/reports/upload-photo', {
            method: 'POST',
            headers: {
                'Authorization': 'Bearer ' + localStorage.getItem('token')
            },
            body: formData
        });
        
        const result = await response.json();
        photos.value.push(result.data.url);
    }
};
</script>
```

## ⚠️ 注意事项

### 文件格式限制

| 文件类型 | 支持格式 |
| -------- | -------- |
| 头像 | jpg, jpeg, png |
| 现场图片 | jpg, jpeg, png, gif, webp, bmp |

### 文件大小限制

- 默认最大文件大小：10MB
- 如需调整，可在 `application.yml` 中配置：
  ```yaml
  spring:
    servlet:
      multipart:
        max-file-size: 10MB
        max-request-size: 10MB
  ```

### 图片访问权限

- **上传接口**：需要登录认证（携带 JWT Token）
- **访问接口**：公开访问（无需认证），任何人都可以通过 URL 访问图片

### 图片命名规则

- 文件名格式：`{时间戳}.{扩展名}`
- 示例：`1704758400000.jpg`

### 多张图片上传

现场图片支持多张上传，需多次调用上传接口，然后将返回的 URL 用逗号分隔传入上报接口的 `photos` 字段：

```json
{
    "photos": "/api/file/report/1704758400001.jpg,/api/file/report/1704758400002.jpg"
}
```
