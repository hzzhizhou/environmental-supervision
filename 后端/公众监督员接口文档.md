# 公众监督员接口文档

***

## 🌐 服务基础信息

| 属性               | 值                       |
| ---------------- | ----------------------- |
| **服务地址**         | `http://localhost:8080` |
| **API前缀**        | `/api/supervisor`       |
| **认证方式**         | JWT Token               |
| **Content-Type** | `application/json`      |

***

## 🔐 认证说明

所有公众监督员接口均需要登录认证，需在请求头中携带 JWT Token：

```bash
Authorization: Bearer <token>
```

**Token 获取方式**：通过 `/api/auth/login` 登录接口获取

## 2. 获取城市列表

### 请求信息

| 属性         | 值                                   |
| ---------- | ----------------------------------- |
| **URL**    | `/api/supervisor/regions/cities`    |
| **Method** | GET                                 |
| **认证**     | 需要                                  |

### 请求参数

| 字段       | 类型      | 必填 | 说明       |
| -------- | ------- | -- | -------- |
| provinceId | Integer | ✅  | 省份ID |

### 请求示例

```bash
GET /api/supervisor/regions/cities?provinceId=6
Authorization: Bearer <token>
```

### 响应示例

```json
{
    "code": 200,
    "message": "获取成功",
    "data": [
        {
            "cityId": 1,
            "cityCode": "2101",
            "cityName": "沈阳市",
            "provinceId": 6
        },
        {
            "cityId": 2,
            "cityCode": "2102",
            "cityName": "大连市",
            "provinceId": 6
        },
        {
            "cityId": 3,
            "cityCode": "2103",
            "cityName": "鞍山市",
            "provinceId": 6
        }
    ]
}
```

### 使用说明

此接口用于实现省份-城市联动选择功能：
1. 先调用 `/api/supervisor/regions/provinces` 获取所有省份
2. 用户选择省份后，将该省份的 `provinceId` 传入此接口获取对应城市列表
3. 城市列表会根据选择的省份动态更新

## 3. 现场图片上传

### 请求信息

| 属性               | 值                                      |
| ---------------- | -------------------------------------- |
| **URL**          | `/api/supervisor/reports/upload-photo` |
| **Method**       | POST                                   |
| **认证**           | 需要（携带JWT Token）                        |
| **Content-Type** | `multipart/form-data`                  |

### 请求参数

| 字段   | 类型            | 必填 | 说明 |
| ---- | ------------- | -- | -- |
| file | MultipartFile | ✅  | 图片文件（支持 jpg/png/jpeg/gif/webp/bmp 格式） |

### 请求示例

```bash
POST /api/supervisor/reports/upload-photo
Content-Type: multipart/form-data
Authorization: Bearer <token>

# 表单字段：
# file: (选择本地图片文件)
```

### 响应示例

```json
{
    "code": 200,
    "message": "图片上传成功",
    "data": {
        "url": "/api/file/report/1781094089812.png"
    }
}
```

### 查看图片

上传成功后，可通过以下 URL 访问图片（无需认证）：

```
GET /api/file/report/1781094089812.png
```

### 说明

- 支持单张图片上传
- 图片存储在服务器 `uploads/report/` 目录下
- 返回的 URL 可直接用于上报接口的 `photos` 字段
- 如需上传多张图片，请多次调用此接口

## 4. 上报污染物

### 请求信息

| 属性         | 值                         |
| ---------- | ------------------------- |
| **URL**    | `/api/supervisor/reports` |
| **Method** | POST                      |
| **认证**     | 需要                        |

### 请求参数

| 字段          | 类型      | 必填 | 说明                             |
| ----------- | ------- | -- | ------------------------------ |
| pollutionType | String  | ✅  | 污染类型                           |
| pollutionLevel | String  | ✅  | 污染程度（优/良/轻度污染/中度污染/重度污染/严重污染） |

| address     | String  | ❌  | 详细地址                           |

## 7. 删除上报

### 请求信息

| 属性         | 值                                    |
| ---------- | ------------------------------------ |
| **URL**    | `/api/supervisor/reports/{reportId}` |
| **Method** | DELETE                               |
| **认证**     | 需要                                   |

### 请求参数

| 字段       | 类型      | 必填 | 说明                     |
| -------- | ------- | -- | ---------------------- |
| reportId | Integer | ✅  | 上报ID（路径参数）             |

### 请求示例

```bash
DELETE /api/supervisor/reports/1
Authorization: Bearer <token>
```

### 响应示例

**成功响应：**

```json
{
    "code": 200,
    "message": "删除成功",
    "data": null
}
```

**错误响应：**

```json
{
    "code": 400,
    "message": "上报记录不存在",
    "data": null
}
```

```json
{
    "code": 403,
    "message": "无权删除此上报记录",
    "data": null
}
```

### 注意事项

- 用户只能删除自己创建的上报记录
- 如果上报记录已被指派（status=assigned）或已检测（status=detected），可能无法删除（取决于业务规则）

## 💡 使用流程

```
1. 用户注册 → 默认成为公众监督员
           ↓
2. 用户登录 → 获取 JWT Token
           ↓
3. 携带 Token → 调用各接口
           ↓
4. 上传现场图片 → 获取图片URL（可多次上传）
           ↓
5. 上报污染物（携带图片URL） → 获得20积分
           ↓
6. 查看积分/排行榜 → 参与排名
```
