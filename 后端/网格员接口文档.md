# 网格员接口文档

## 概述

网格员接口提供任务管理和检测数据提交功能。网格员可以查看指派给自己的任务、接受任务、提交检测数据等操作。

**接口前缀**: `/api/grid-worker`

**认证方式**: JWT Token，需在请求头中携带 `Authorization: Bearer <token>`

---

## 一、任务管理接口

### 1.1 获取任务列表

| 属性 | 值 |
|------|-----|
| **方法** | GET |
| **路径** | `/api/grid-worker/tasks` |
| **权限** | 网格员 |

**请求参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| status | String | ❌ | 状态筛选（assigned/accepted/completed） |
| pageNum | Integer | ❌ | 页码（默认1） |
| pageSize | Integer | ❌ | 每页数量（默认10） |

**状态说明**:

| 状态码 | 状态描述 | 说明 |
|--------|----------|------|
| assigned | 已指派 | 管理员已指派任务，等待网格员接受 |
| accepted | 已接受 | 网格员已接受任务 |
| completed | 已完成 | 网格员已提交检测数据 |

**响应示例**:
```json
{
    "code": 200,
    "message": "success",
    "data": {
        "tasks": {
            "records": [
                {
                    "taskId": 1,
                    "reportId": 1,
                    "reportCode": "AQI-20260610-001",
                    "userId": 2,
                    "userName": "网格员张三",
                    "adminId": 3,
                    "adminName": "管理员1张三",
                    "assignTime": "2026-06-10T22:00:00",
                    "deadline": "2026-06-11T22:00:00",
                    "status": "assigned",
                    "statusDesc": "已指派",
                    "rejectReason": null
                }
            ],
            "total": 12,
            "size": 10,
            "current": 1,
            "pages": 2
        },
        "statistics": {
            "total": 12,
            "assigned": 3,
            "accepted": 5,
            "finish": 4
        }
    }
}
```

**响应字段说明**:

| 字段 | 类型 | 说明 |
|------|------|------|
| tasks | Object | 任务列表（分页） |
| tasks.records | Array | 任务记录数组 |
| tasks.total | Integer | 总记录数 |
| tasks.size | Integer | 每页数量 |
| tasks.current | Integer | 当前页码 |
| tasks.pages | Integer | 总页数 |
| statistics | Object | 状态统计 |
| statistics.total | Integer | 总任务数 |
| statistics.assigned | Integer | 已指派任务数 |
| statistics.accepted | Integer | 已接受任务数 |
| statistics.finish | Integer | 已完成任务数 |

### 1.4 提交检测数据

| 属性 | 值 |
|------|-----|
| **方法** | POST |
| **路径** | `/api/grid-worker/detect-data` |
| **权限** | 网格员 |

**请求体**:

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| reportId | Integer | ✅ | 上报ID（**注意**：不需要传 taskId，后端会根据 reportId 自动关联） |
| aqiValue | Integer | ✅ | 实测AQI数值 |
| aqiLevel | String | ✅ | 实测AQI等级（优/良/轻度污染/中度污染/重度污染/严重污染） |
| pm25 | BigDecimal | ✅ | 实测PM2.5浓度 |
| pm10 | BigDecimal | ✅ | 实测PM10浓度 |
| o3 | BigDecimal | ✅ | 实测臭氧浓度 |
| so2 | BigDecimal | ✅ | 实测二氧化硫浓度 |
| no2 | BigDecimal | ✅ | 实测二氧化氮浓度 |
| co | BigDecimal | ✅ | 实测一氧化碳浓度 |
| location | String | ✅ | 检测地点 |
| remark | String | ❌ | 检测备注 |

**请求示例**:
```json
{
    "reportId": 1,
    "aqiValue": 156,
    "aqiLevel": "中度污染",
    "pm25": 125.5,
    "pm10": 180.3,
    "o3": 85.2,
    "so2": 15.0,
    "no2": 45.5,
    "co": 1.2,
    "location": "和平区文化路XX号",
    "remark": "现场检测完成，空气质量较差"
}
```

**响应示例**:
```json
{
    "code": 200,
    "message": "检测数据提交成功",
    "data": {
        "detectId": 1,
        "taskId": 1,
        "reportCode": "AQI-20260610-001",
        "userId": 2,
        "userName": "网格员张三",
        "detectTime": "2026-06-10T23:00:00",
        "aqiValue": 156,
        "aqiLevel": "中度污染",
        "pm25": 125.5,
        "pm10": 180.3,
        "o3": 85.2,
        "so2": 15.0,
        "no2": 45.5,
        "co": 1.2,
        "photos": null,
        "confirmStatus": 0,
        "confirmStatusDesc": "待审核",
        "confirmedAt": null
    }
}
```

## 二、排行榜接口

### 2.1 获取网格员排行榜

| 属性 | 值 |
|------|-----|
| **方法** | GET |
| **路径** | `/api/grid-worker/ranking` |
| **权限** | 网格员 |

**请求参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| pageNum | Integer | ❌ | 页码（默认1） |
| pageSize | Integer | ❌ | 每页数量（默认10） |

**响应示例**:
```json
{
    "code": 200,
    "message": "success",
    "data": {
        "records": [
            {
                "rank": 1,
                "userId": 5,
                "username": "worker001",
                "realName": "网格员张三",
                "gridName": "和平区网格001",
                "completedTasks": 15,
                "totalTasks": 18
            },
            {
                "rank": 2,
                "userId": 6,
                "username": "worker002",
                "realName": "网格员李四",
                "gridName": "沈河区网格001",
                "completedTasks": 12,
                "totalTasks": 15
            }
        ],
        "total": 10,
        "size": 10,
        "current": 1,
        "pages": 1,
        "myRank": 1,
        "myCompletedTasks": 15
    }
}
```

**响应字段说明**:

| 字段 | 类型 | 说明 |
|------|------|------|
| records | Array | 排行榜记录列表 |
| records[].rank | Integer | 排名 |
| records[].userId | Integer | 用户ID |
| records[].username | String | 登录账号 |
| records[].realName | String | 真实姓名 |
| records[].gridName | String | 所属网格 |
| records[].completedTasks | Integer | 已完成任务数 |
| records[].totalTasks | Integer | 总任务数 |
| total | Long | 总记录数 |
| size | Integer | 每页数量 |
| current | Integer | 当前页码 |
| pages | Integer | 总页数 |
| myRank | Integer | 当前用户排名 |
| myCompletedTasks | Integer | 当前用户完成数 |

**说明**:
- 排行榜根据完成任务数量降序排列
- 当前用户可以看到自己的排名和完成任务数
- 只统计角色为网格员（roleId=2）的用户

## 六、使用示例

### cURL 示例

```bash
# 获取任务列表
curl -X GET "http://localhost:8080/api/grid-worker/tasks?pageNum=1&pageSize=10" \
  -H "Authorization: Bearer <token>"

# 接受任务
curl -X PUT "http://localhost:8080/api/grid-worker/tasks/1/accept" \
  -H "Authorization: Bearer <token>"

# 提交检测数据
curl -X POST "http://localhost:8080/api/grid-worker/detect-data" \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "reportId": 1,
    "aqiValue": 156,
    "aqiLevel": "中度污染",
    "pm25": 125.5,
    "pm10": 180.3,
    "o3": 85.2,
    "so2": 15.0,
    "no2": 45.5,
    "co": 1.2,
    "location": "和平区文化路XX号"
  }'

# 获取检测数据列表
curl -X GET "http://localhost:8080/api/grid-worker/detect-data?confirmStatus=0" \
  -H "Authorization: Bearer <token>"
```
