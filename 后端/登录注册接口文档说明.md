# 登录注册接口文档

---

## 🌐 服务基础信息

| 属性 | 值 |
|------|-----|
| **服务地址** | `http://localhost:8080` |
| **API前缀** | `/api/auth` |
| **Content-Type** | `application/json` |
| **字符编码** | UTF-8 |

---

## 🔐 1. 用户注册接口

### 接口信息

| 属性 | 值 |
|------|-----|
| **URL** | `/api/auth/register` |
| **Method** | POST |
| **认证方式** | 无需认证 |

### 请求参数

| 字段名 | 类型 | 必填 | 长度限制 | 说明 |
|--------|------|------|----------|------|
| username | String | ✅ | 3-20 | 登录账号，需唯一 |
| password | String | ✅ | 6-50 | 登录密码（明文传递） |
| realName | String | ✅ | - | 用户昵称 |
| phone | String | ✅ | 11位 | 手机号码，需唯一 |
| code | String | ✅ | 6位 | 验证码，固定为 `123456` |

### 请求示例

```json
POST /api/auth/register
Content-Type: application/json

{
    "username": "public001",
    "password": "123456",
    "realName": "张三",
    "phone": "13800138000",
    "code": "123456"
}
```

### 成功响应

| 字段名 | 类型 | 说明 |
|--------|------|------|
| code | Integer | 响应状态码，成功为200 |
| message | String | 响应消息 |
| data | Object | 用户信息 |
| data.userId | Integer | 用户ID |
| data.username | String | 用户名 |
| data.realName | String | 真实姓名 |
| data.phone | String | 手机号 |
| data.roleId | Integer | 角色ID（默认为3） |
| data.roleName | String | 角色名称（公众监督员） |
| data.status | Integer | 账号状态（1=正常） |
| data.statusDesc | String | 状态描述 |
| data.createdAt | String | 注册时间 |

**响应示例：**

```json
{
    "code": 200,
    "message": "注册成功",
    "data": {
        "userId": 1,
        "username": "public001",
        "realName": "张三",
        "phone": "13800138000",
        "roleId": 3,
        "roleName": "公众监督员",
        "gridId": null,
        "avatarUrl": null,
        "status": 1,
        "statusDesc": "正常",
        "lastLoginAt": null,
        "createdAt": "2026-06-06T14:00:00"
    }
}
```

### 错误响应

| 错误码 | 错误信息 | 原因 |
|--------|----------|------|
| 400 | `参数校验失败` | 必填字段为空或格式错误 |
| 409 | `用户名 'xxx' 已被注册` | 用户名重复 |
| 409 | `手机号 'xxx' 已被注册` | 手机号重复 |
| 500 | `验证码错误` | 验证码不正确（必须为123456） |

## 🔑 2. 用户登录接口

### 接口信息

| 属性 | 值 |
|------|-----|
| **URL** | `/api/auth/login` |
| **Method** | POST |
| **认证方式** | 无需认证 |

### 请求参数

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| username | String | ✅ | 用户名 |
| password | String | ✅ | 密码（明文传递） |
| roleId | Integer | ❌ | 角色ID，用于角色判断 |

### 请求示例

**方式一：不指定角色（自动识别）**

```json
POST /api/auth/login
Content-Type: application/json

{
    "username": "public001",
    "password": "123456"
}
```

**方式二：指定角色登录**

```json
POST /api/auth/login
Content-Type: application/json

{
    "username": "admin",
    "password": "123456",
    "roleId": 1
}
```

### 成功响应

| 字段名 | 类型 | 说明 |
|--------|------|------|
| code | Integer | 响应状态码，成功为200 |
| message | String | 响应消息 |
| data | Object | 登录信息 |
| data.userId | Integer | 用户ID |
| data.username | String | 用户名 |
| data.realName | String | 真实姓名 |
| data.phone | String | 手机号 |
| data.roleId | Integer | 角色ID |
| data.roleName | String | 角色名称 |
| data.gridId | Integer | 网格ID（如有） |
| data.gridName | String | 网格名称（如有） |
| data.token | String | JWT令牌 |
| data.expireTime | String | 令牌过期时间 |

**响应示例：**

```json
{
    "code": 200,
    "message": "登录成功",
    "data": {
        "userId": 1,
        "username": "public001",
        "realName": "张三",
        "phone": "13800138000",
        "roleId": 3,
        "roleName": "公众监督员",
        "gridId": null,
        "gridName": null,
        "token": "eyJhbGciOiJIUzI1NiJ9.eyJzdWIiOiIxIiwidXNlcm5hbWUiOiJwdWJsaWMwMDEiLCJyb2xlSWQiOjMsImlhdCI6MTc4MDY2MDA0NiwiZXhwIjoxNzgwNzQ2NDQ2fQ.rx1SrQsgDQnqkAfvZoruBCed_24Ww6isvT403V_yh-Q",
        "expireTime": "2026-06-07T14:00:00"
    }
}
```

### 错误响应

| 错误码 | 错误信息 | 原因 |
|--------|----------|------|
| 400 | `参数校验失败` | 必填字段为空 |
| 401 | `用户名不存在` | 用户未注册 |
| 401 | `密码错误` | 密码不正确 |
| 401 | `该用户不属于所选角色` | 指定的roleId与用户实际角色不符 |
| 500 | `用户已被禁用` | 账号状态为禁用 |

## 🚪 3. 用户退出接口

### 接口信息

| 属性 | 值 |
|------|-----|
| **URL** | `/api/auth/logout` |
| **Method** | POST |
| **认证方式** | JWT Token |

### 请求头

| 字段名 | 值 |
|--------|-----|
| Authorization | `Bearer {token}` |

### 请求参数

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| userId | Integer | ✅ | 用户ID |

### 请求示例

```json
POST /api/auth/logout
Content-Type: application/json
Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...

{
    "userId": 1
}
```

### 成功响应

```json
{
    "code": 200,
    "message": "退出成功",
    "data": null
}
```

## 🎭 角色ID对照表

| roleId | 角色名称 | 角色代码 | 说明 |
|--------|----------|----------|------|
| 1 | 系统管理员 | ADMIN | 最高权限，可管理所有功能 |
| 2 | 网格员 | WORKER | 负责现场检测数据采集 |
| 3 | 公众监督员 | PUBLIC | 普通用户，可上报AQI数据 |
| 4 | 决策者 | DECIDER | 查看统计分析数据 |

---

## 🔓 角色获取方式

| 角色 | 获取方式 | 说明 |
|------|----------|------|
| 公众监督员 | ✅ **注册接口** | 普通用户注册默认角色 |
| 系统管理员 | ❌ **管理后台** | 需管理员创建 |
| 网格员 | ❌ **管理后台** | 需管理员创建并分配网格 |
| 决策者 | ❌ **管理后台** | 需管理员创建 |

## 💡 使用说明

1. **注册**：普通用户只能注册为公众监督员（roleId=3），其他角色需管理员创建
2. **登录**：支持两种方式，不指定角色自动识别，指定角色则进行角色校验
3. **验证码**：当前为固定值 `123456`，生产环境需替换为动态验证码
4. **密码安全**：密码采用BCrypt加密存储，前端传输时建议使用HTTPS
