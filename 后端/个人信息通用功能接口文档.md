# 个人信息通用功能接口文档

---

## 🌐 服务基础信息

| 属性 | 值 |
|------|-----|
| **服务地址** | `http://localhost:8080` |
| **API前缀** | `/api/profile` |
| **认证方式** | JWT Token |
| **Content-Type** | `application/json` |

---

## 🔐 认证说明

所有个人中心接口均需要登录认证，需在请求头中携带 JWT Token：

```bash
Authorization: Bearer <token>
```

**Token 获取方式**：通过 `/api/auth/login` 登录接口获取

---

## 📋 接口列表

| 序号 | 接口 | Method | 描述 |
|------|------|--------|------|
| 1 | `/api/profile/info/{userId}` | GET | 获取个人信息 |
| 2 | `/api/profile/update` | POST | 修改个人信息（昵称/手机号） |
| 3 | `/api/profile/change-password` | POST | 修改密码 |
| 4 | `/api/profile/upload-avatar` | POST | 上传头像 |
| 5 | `/api/profile/avatar/{filename}` | GET | 访问头像 |

---

## 1. 获取个人信息

### 请求信息

| 属性 | 值 |
|------|-----|
| **URL** | `/api/profile/info/{userId}` |
| **Method** | GET |
| **认证** | 需要 |

### 路径参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| userId | Integer | ✅ | 用户ID |

### 响应示例

```json
{
    "code": 200,
    "message": "获取成功",
    "data": {
        "userId": 1,
        "username": "public001",
        "realName": "张三",
        "phone": "13800138000",
        "roleId": 3,
        "roleName": "公众监督员",
        "gridId": null,
        "gridName": null,
        "avatarUrl": null,
        "status": 1,
        "statusDesc": "正常",
        "lastLoginAt": "2026-06-06T15:00:00",
        "createdAt": "2026-06-01T10:00:00"
    }
}
```

### 响应字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| userId | Integer | 用户ID |
| username | String | 用户名（不可修改） |
| realName | String | 昵称（可修改） |
| phone | String | 手机号（可修改） |
| roleId | Integer | 角色ID |
| roleName | String | 角色名称 |
| gridId | Integer | 网格ID（仅网格员有值） |
| gridName | String | 网格名称 |
| avatarUrl | String | 头像URL |
| status | Integer | 账号状态（1=正常，0=禁用） |
| statusDesc | String | 状态描述 |
| lastLoginAt | String | 最后登录时间 |
| createdAt | String | 创建时间 |

---

## 2. 修改个人信息

### 请求信息

| 属性 | 值 |
|------|-----|
| **URL** | `/api/profile/update` |
| **Method** | POST |
| **认证** | 需要 |

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| userId | Integer | ✅ | 用户ID |
| realName | String | ❌ | 昵称（真实姓名） |
| phone | String | ❌ | 手机号（11位） |

### 请求示例

```json
{
    "userId": 1,
    "realName": "李四",
    "phone": "13900139000"
}
```

### 响应示例

```json
{
    "code": 200,
    "message": "个人信息修改成功",
    "data": {
        "userId": 1,
        "username": "public001",
        "realName": "李四",
        "phone": "13900139000",
        "roleId": 3,
        "roleName": "公众监督员"
    }
}
```

### 错误响应

| 错误码 | 错误信息 | 原因 |
|--------|----------|------|
| 400 | 参数校验失败 | 必填字段为空或格式错误 |
| 500 | 用户不存在 | userId对应的用户不存在 |
| 409 | 手机号 'xxx' 已被注册 | 新手机号已被其他用户使用 |

## 3. 修改密码

### 请求信息

| 属性 | 值 |
|------|-----|
| **URL** | `/api/profile/change-password` |
| **Method** | POST |
| **认证** | 需要 |

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| userId | Integer | ✅ | 用户ID |
| oldPassword | String | ✅ | 旧密码 |
| newPassword | String | ✅ | 新密码（6-50字符） |

### 请求示例

```json
{
    "userId": 1,
    "oldPassword": "123456",
    "newPassword": "654321"
}
```

### 响应示例

```json
{
    "code": 200,
    "message": "密码修改成功",
    "data": null
}
```

### 错误响应

| 错误码 | 错误信息 | 原因 |
|--------|----------|------|
| 400 | 参数校验失败 | 必填字段为空或密码长度不符合要求 |
| 500 | 用户不存在 | userId对应的用户不存在 |
| 500 | 旧密码错误 | 旧密码输入不正确 |

## 4. 上传头像

### 请求信息

| 属性 | 值 |
|------|-----|
| **URL** | `/api/profile/upload-avatar` |
| **Method** | POST |
| **认证** | 需要 |
| **Content-Type** | `multipart/form-data` |

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| userId | Integer | ✅ | 用户ID |
| file | MultipartFile | ✅ | 头像文件 |

### 请求示例

```bash
POST /api/profile/upload-avatar
Content-Type: multipart/form-data

Form Data:
- userId: 1
- file: [选择图片文件]
```

### 响应示例

```json
{
    "code": 200,
    "message": "头像上传成功",
    "data": "/api/profile/avatar/1620000000000.jpg"
}
```

### 错误响应

| 错误码 | 错误信息 | 原因 |
|--------|----------|------|
| 500 | 用户不存在 | userId对应的用户不存在 |
| 500 | 头像上传失败 | 文件上传过程出错 |

---

## 5. 访问头像

### 请求信息

| 属性 | 值 |
|------|-----|
| **URL** | `/api/profile/avatar/{filename}` |
| **Method** | GET |
| **认证** | 不需要 |

### 路径参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| filename | String | ✅ | 头像文件名 |

### 响应示例

```
返回图片二进制数据（JPEG格式）
```

### 错误响应

| 错误码 | 错误信息 | 原因 |
|--------|----------|------|
| 500 | 头像不存在 | 指定的头像文件不存在 |

## 🎭 角色说明

所有角色均可以使用个人中心接口，接口会根据用户角色返回相应的信息：

| roleId | 角色名称 | 说明 |
|--------|----------|------|
| 1 | 系统管理员 | 可查看所有信息，无网格 |
| 2 | 网格员 | 可查看网格信息 |
| 3 | 公众监督员 | 默认注册角色，无网格 |
| 4 | 决策者 | 可查看所有信息，无网格 |

---

## 💡 使用流程

```
1. 用户登录 → 获取 JWT Token
           ↓
2. 携带 Token → 调用个人中心接口
           ↓
3. 获取/修改个人信息 → 返回操作结果
```
