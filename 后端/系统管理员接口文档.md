# 系统管理员接口文档

## 概述

系统管理员接口提供对用户、网格、上报记录、任务指派和检测数据的管理功能。所有接口都需要管理员权限认证。

**接口前缀**: `/api/admin`

**认证方式**: JWT Token，需在请求头中携带 `Authorization: Bearer <token>`

---

## 一、登录接口

### 1.1 用户登录

| 属性 | 值 |
|------|-----|
| **方法** | POST |
| **路径** | `/api/auth/login` |
| **权限** | 公开 |

**请求体**:

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | String | ✅ | 登录账号 |
| password | String | ✅ | 密码 |
| roleId | Integer | ❌ | 角色ID（可选，用于验证角色） |

**请求示例**:
```json
{
    "username": "admin",
    "password": "123456"
}
```

**说明**:
- `roleId` 是可选参数，不传时系统自动识别用户角色
- 如果传了 `roleId`，会验证用户是否属于该角色（1-管理员，2-网格员，3-公众监督员，4-决策者）

**响应示例**:
```json
{
    "code": 200,
    "message": "登录成功",
    "data": {
        "userId": 1,
        "username": "admin",
        "realName": "管理员",
        "phone": "13800138000",
        "roleId": 1,
        "roleName": "系统管理员",
        "gridId": null,
        "gridName": null,
        "token": "eyJhbGciOiJIUzI1NiJ9...",
        "expireTime": "2026-06-11T10:00:00"
    }
}
```

**说明**:
- 管理员、网格员、公众监督员共用此登录接口
- 系统根据用户的 `roleId` 自动识别角色
- 登录成功后返回的 `token` 需在后续请求中携带

**业务规则**:
- 本市有可用网格员时，只能指派本市人员，否则返回错误："本市存在可用网格员，请优先指派本市人员"
- 本市无可用网格员时，方可跨省指派
